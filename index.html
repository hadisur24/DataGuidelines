🔄 Power Query (M Language) — Data Transformation
🎯
Why Power Query?

Power Query is the most powerful data cleaning tool in Excel. It records your transformation steps and auto-refreshes when data changes. This is a MUST-KNOW skill.

Data Cleaning Steps
  • Remove duplicates
  • Handle null/empty values
  • Trim whitespace
  • Change data types
  • Split/merge columns
  • Filter rows by condition
  • Replace specific values
Advanced Transformations
  • Append/Merge queries (like SQL JOIN)
  • Pivot/Unpivot columns
  • Group By (aggregate)
  • Add custom columns (M formula)
  • Conditional column
  • Date table creation
  • Parameters & functions
📄 power-query-m.pq 
// Power Query M Language Examples

// Clean and transform employee data
let
    Source = Excel.Workbook(File.Contents("C:\data\employees.xlsx")),
    Sheet1 = Source{[Name="Sheet1"]}[Data],
    
    // Promote first row as headers
    Headers = Table.PromoteHeaders(Sheet1),
    
    // Change data types
    Types = Table.TransformColumnTypes(Headers,{
        {"Salary", Int64.Type},
        {"HireDate", type date},
        {"Name", type text}
    }),
    
    // Remove rows where salary is null
    NoNulls = Table.SelectRows(Types, each [Salary] <> null),
    
    // Add a new column: Salary Category
    AddCategory = Table.AddColumn(NoNulls, "SalaryTier", each
        if [Salary] >= 100000 then "Senior"
        else if [Salary] >= 60000 then "Mid-Level"
        else "Junior"
    ),
    
    // Trim all text columns
    TrimNames = Table.TransformColumns(AddCategory, {
        {"Name", Text.Trim},
        {"Department", Text.Trim}
    }),
    
    // Extract Year from Date
    AddYear = Table.AddColumn(TrimNames, "HireYear", each Date.Year([HireDate]))
    
in
    AddYear

🗃️ Phase 2: Data Modeling (Star Schema) ▼
⭐
Star Schema is the Gold Standard

Always use a Star Schema: one Fact Table (transactions) connected to multiple Dimension Tables (dates, products, customers, employees). Never create relationships between dimension tables.

Table TypeContainsExample
Fact TableNumeric measures, foreign keysSales, Orders, Revenue
Dim: DateAll date attributesDate, Month, Quarter, Year, Weekday
Dim: CustomerCustomer infoName, City, Segment, Age
Dim: ProductProduct detailsName, Category, Brand, Price
Dim: RegionGeographic dataCountry, State, City, Region
  • Create relationships (1:many direction)
  • Always connect to a Date Dimension table
  • Avoid bidirectional relationships
  • Understand filter context flow
  • Hide foreign key columns from reports

// ═══ BASIC MEASURES ═══
Total Sales = SUM(Sales[Amount])
Total Orders = COUNTROWS(Orders)
Avg Order Value = DIVIDE([Total Sales], [Total Orders], 0)
Unique Customers = DISTINCTCOUNT(Sales[CustomerID])

// ═══ CALCULATE (Heart of DAX) ═══
// CALCULATE changes the filter context
IT Sales = CALCULATE([Total Sales], Employees[Department] = "IT")
High Value = CALCULATE([Total Sales], Sales[Amount] > 10000)
YTD Sales = CALCULATE([Total Sales], DATESYTD(Dates[Date]))

// ═══ TIME INTELLIGENCE ═══
Sales Last Month = CALCULATE([Total Sales], DATEADD(Dates[Date], -1, MONTH))
Sales Last Year = CALCULATE([Total Sales], SAMEPERIODLASTYEAR(Dates[Date]))
MoM Growth = DIVIDE([Total Sales] - [Sales Last Month], [Sales Last Month])
YoY Growth % = DIVIDE([Total Sales] - [Sales Last Year], [Sales Last Year])
MTD Sales = TOTALMTD([Total Sales], Dates[Date])
QTD Sales = TOTALQTD([Total Sales], Dates[Date])

// ═══ ITERATOR FUNCTIONS (Row Context) ═══
Total Revenue = SUMX(Sales, Sales[Quantity] * Sales[UnitPrice])
Weighted Avg Price = DIVIDE(
    SUMX(Sales, Sales[Quantity] * Sales[Price]),
    SUM(Sales[Quantity])
)

// ═══ FILTER FUNCTIONS ═══
Top Products = CALCULATE(
    [Total Sales],
    TOPN(10, Products, [Total Sales], DESC)
)

Sales excl Returns = CALCULATE(
    [Total Sales],
    FILTER(Sales, Sales[OrderType] <> "Return")
)

// ═══ ALL / ALLEXCEPT ═══
Market Share = DIVIDE(
    [Total Sales],
    CALCULATE([Total Sales], ALL(Products))
)

Dept Share = DIVIDE(
    [Total Sales],
    CALCULATE([Total Sales], ALLEXCEPT(Employees, Employees[Department]))
)

// ═══ RANKING ═══
Product Rank = RANKX(ALL(Products), [Total Sales], , DESC, Dense)

// ═══ VARIABLES (Best Practice) ═══
Profit Margin % =
VAR TotalRevenue = SUM(Sales[Revenue])
VAR TotalCost = SUM(Sales[Cost])
VAR Profit = TotalRevenue - TotalCost
RETURN
    DIVIDE(Profit, TotalRevenue)

📄 eda-template.py 
import pandas as pd, numpy as np
import matplotlib.pyplot as plt, seaborn as sns

# ══ STEP 1: LOAD & INITIAL LOOK ══
df = pd.read_csv('dataset.csv')
print(f"Shape: {df.shape}")
print(df.head())
print(df.info())
print(df.describe())

# ══ STEP 2: MISSING VALUES ══
missing = df.isnull().sum().sort_values(ascending=False)
missing_pct = (missing / len(df) * 100).round(2)
missing_report = pd.DataFrame({'Count': missing, '%': missing_pct})
print(missing_report[missing_report['Count'] > 0])

# ══ STEP 3: DUPLICATES ══
print(f"Duplicates: {df.duplicated().sum()}")
df = df.drop_duplicates()

# ══ STEP 4: OUTLIER DETECTION ══
def detect_outliers(df, col):
    Q1 = df[col].quantile(0.25)
    Q3 = df[col].quantile(0.75)
    IQR = Q3 - Q1
    outliers = df[(df[col] < Q1-1.5*IQR) | (df[col] > Q3+1.5*IQR)]
    print(f"{col}: {len(outliers)} outliers ({len(outliers)/len(df)*100:.1f}%)")
    return outliers

# ══ STEP 5: DISTRIBUTIONS ══
num_cols = df.select_dtypes(include=np.number).columns
df[num_cols].hist(figsize=(12,8), bins=20)
plt.suptitle('Numerical Distributions')

# ══ STEP 6: CORRELATION ══
plt.figure(figsize=(10,6))
sns.heatmap(df[num_cols].corr(), annot=True, cmap='coolwarm', fmt='.2f')
plt.title('Correlation Matrix')

# ══ STEP 7: CATEGORY ANALYSIS ══
cat_cols = df.select_dtypes(include='object').columns
for col in cat_cols:
    print(f"\n{col}:")
    print(df[col].value_counts())

# ══ STEP 8: KEY INSIGHTS ══
insights = {
    "Total Records": len(df),
    "Avg Salary": df["salary"].mean().round(2),
    "Top Department": df["department"].mode()[0],
    "Correlation(exp,salary)": df[["experience","salary"]].corr().iloc[0,1].round(3)
}
print("\n📊 KEY INSIGHTS:")
for k, v in insights.items():
    print(f"  → {k}: {v}")

-- ✅ JOIN with aggregation
SELECT
    d.dept_name,
    COUNT(e.emp_id) AS headcount,
    ROUND(AVG(e.salary), 2) AS avg_salary,
    SUM(e.salary) AS total_payroll
FROM employees e
JOIN departments d ON e.dept_id = d.dept_id
GROUP BY d.dept_name
HAVING COUNT(e.emp_id) > 5
ORDER BY avg_salary DESC;

SELECT first_name, salary,

        WHEN salary >= 65000  THEN '🥈 Mid'
        ELSE '🥉 Junior'
    END AS level
FROM employees;
🟡 Level 3: Window Functions (⭐ Interview Essential)▼
📄 window-functions.sql
-- Ranking within departments
SELECT first_name, department, salary,
    ROW_NUMBER() OVER(PARTITION BY department ORDER BY salary DESC) AS row_num,
    RANK() OVER(PARTITION BY department ORDER BY salary DESC) AS rank_num,
    DENSE_RANK() OVER(PARTITION BY department ORDER BY salary DESC) AS dense_rnk
FROM employees;

-- LAG & LEAD (MoM comparison)
SELECT month, revenue,
    LAG(revenue) OVER(ORDER BY month) AS prev_month,
    revenue - LAG(revenue) OVER(ORDER BY month) AS change
FROM monthly_sales;

-- Running Total & Moving Average
SELECT sale_date, revenue,
    SUM(revenue) OVER(ORDER BY sale_date) AS running_total,
    AVG(revenue) OVER(ORDER BY sale_date ROWS BETWEEN 6 PRECEDING AND CURRENT ROW) AS moving_7day
FROM daily_sales;

-- ✅ CTE + Window: Top 3 per Department
WITH ranked AS (
    SELECT *, DENSE_RANK() OVER(PARTITION BY dept ORDER BY salary DESC) AS rnk
    FROM employees
)
SELECT * FROM ranked WHERE rnk <= 3;

-- ✅ Year-over-Year Growth
WITH yearly AS (
    SELECT YEAR(order_date) AS yr, SUM(amount) AS revenue
    FROM orders GROUP BY YEAR(order_date)
)
SELECT yr, revenue,
    LAG(revenue) OVER(ORDER BY yr) AS prev_year,
    ROUND((revenue - LAG(revenue) OVER(ORDER BY yr)) * 100.0
           / LAG(revenue) OVER(ORDER BY yr), 2) AS yoy_pct
FROM yearly;

-- ✅ Second Highest Salary
WITH ranked AS (
    SELECT salary, DENSE_RANK() OVER(ORDER BY salary DESC) AS rnk
    FROM employees
)
SELECT DISTINCT salary FROM ranked WHERE rnk = 2;

📄 statistics-python.py 
import pandas as pd, numpy as np
from scipy import stats
import matplotlib.pyplot as plt, seaborn as sns

# ═══ DESCRIPTIVE STATISTICS ═══
salary = df['salary']
print(f"Mean:    {salary.mean():.2f}")
print(f"Median:  {salary.median():.2f}")
print(f"Mode:    {salary.mode()[0]:.2f}")
print(f"Std Dev: {salary.std():.2f}")
print(f"Skew:    {salary.skew():.3f}")  # >0 right skewed

# ═══ OUTLIER DETECTION (IQR Method) ═══
Q1 = salary.quantile(0.25)
Q3 = salary.quantile(0.75)
IQR = Q3 - Q1
lower_bound = Q1 - 1.5 * IQR
upper_bound = Q3 + 1.5 * IQR
outliers = salary[(salary < lower_bound) | (salary > upper_bound)]
print(f"Outliers: {len(outliers)} records")

# ═══ CORRELATION ═══
corr_matrix = df[['salary','experience','performance']].corr()
print(corr_matrix)

# Pearson correlation + p-value
r, p_value = stats.pearsonr(df['experience'], df['salary'])
print(f"r={r:.3f}, p={p_value:.4f}")

# ═══ HYPOTHESIS TESTING ═══
# T-test: Is IT dept salary different from HR?
it_sal = df[df['dept']=='IT']['salary']
hr_sal = df[df['dept']=='HR']['salary']
t_stat, p_val = stats.ttest_ind(it_sal, hr_sal)
print(f"t-statistic: {t_stat:.3f}")
print(f"p-value:     {p_val:.4f}")
print(f"Significant: {p_val < 0.05}")

# ═══ A/B TESTING ═══
# Compare conversion rates: A vs B
group_a = df[df['group']=='A']
group_b = df[df['group']=='B']

conv_a = group_a['converted'].mean()
conv_b = group_b['converted'].mean()
print(f"Conversion A: {conv_a:.2%}")
print(f"Conversion B: {conv_b:.2%}")

# Statistical significance for proportions
n_a, conv_count_a = len(group_a), group_a['converted'].sum()
n_b, conv_count_b = len(group_b), group_b['converted'].sum()
z_stat, p_val = stats.proportions_ztest([conv_count_a, conv_count_b], [n_a, n_b])
print(f"A/B Test p-value: {p_val:.4f} → {'Significant ✅' if p_val < 0.05 else 'Not Significant ❌'}")

# ═══ LINEAR REGRESSION ═══
from sklearn.linear_model import LinearRegression
from sklearn.metrics import r2_score, mean_absolute_error

X = df[['experience', 'age', 'performance']]
y = df['salary']
model = LinearRegression().fit(X, y)
print(f"R² Score: {r2_score(y, model.predict(X)):.3f}")
print(f"MAE: {mean_absolute_error(y, model.predict(X)):.2f}")
📐 Key Statistical Formulas to Know

Descriptive

Mean = Σx / n
Variance = Σ(x - mean)² / (n-1)
Std Dev = √Variance
IQR = Q3 - Q1
Outlier = x < Q1 - 1.5×IQR OR x > Q3 + 1.5×IQR
Z-score = (x - mean) / std_dev

Inferential

CI = mean ± (z × std_error)
std_error = σ / √n
t = (x̄₁ - x̄₂) / √(s²/n₁ + s²/n₂)
Correlation r = Σ(xi-x̄)(yi-ȳ) / √[Σ(xi-x̄)² × Σ(yi-ȳ)²]
p-value < 0.05 → Reject null hypothesis ✅

📖 The Data Storytelling Framework
1️⃣
SITUATION
Set the context. What business problem are we solving? Who is the audience? What data do we have?
"We analyzed 6 months of sales data to understand why Q3 revenue dropped 15%"
2️⃣
INSIGHT
What did the data reveal? Focus on the most important 1-3 findings. Use specific numbers, not vague statements.
"The decline was driven by 3 key factors: product returns (+40%), customer churn in West region, and lower AOV"
3️⃣
ACTION
What should be done? Provide clear, specific, actionable recommendations. Make it easy for stakeholders to decide.
"Recommend: reduce return rate by improving product descriptions, re-engage West region with targeted offers"

🎤 Presentation Skills for Data Analysts

✅ Do These

  • Start with the "so what" — lead with the insight
  • Use executive summary (1 slide with key findings)
  • Explain why it matters to the business
  • Use simple language — avoid jargon
  • Show before/after or comparison context
  • Always end with clear recommendations
  • Anticipate and answer "so what?"
  • Practice presenting out loud before meetings

❌ Avoid These

  • Dumping all data without a story
  • Using too many charts on one slide
  • Saying "the data shows" without explaining what it means
  • Using 3D charts (they distort data)
  • Reading slides word for word
  • Using too many colors (max 3-4)
  • Finishing without clear next steps
  • Assuming audience knows the technical context
📝 Writing Effective Data Reports
📋 Report Structure Template▼
Standard Data Analysis Report Structure:
  1. Executive Summary — Key findings in 3-4 bullet points (write this last)
  2. Business Question — What problem are we solving and why it matters
  3. Data Overview — Sources, time period, limitations, assumptions
  4. Key Findings — Top 3-5 insights with supporting visuals & numbers
  5. Root Cause Analysis — Why is this happening? Dig deeper.
  6. Recommendations — Specific, actionable, prioritized by impact
  7. Next Steps — What should be done, by whom, by when
  8. Appendix — Technical details, methodology, full data tables
💬 Insight Writing Examples▼
❌
Weak Insight (Avoid)

"Revenue was higher in Q4 than Q3. The IT department has more employees. Customer satisfaction scores varied."

✅
Strong Insight (Use This)

"Q4 revenue surged 34% to ₹2.4Cr, driven by the new product launch (contributing 60% of growth). The IT department's 23% headcount increase led to a 15% productivity gain. Customer satisfaction dropped to 3.2/5 in October — correlated with a 3-day delivery delay from our main supplier."

The formula for a strong insight:
[What happened] + [by how much / compared to what] + [why it happened] + [what it means for the business]

🎤 Common Interview Questions & How to Answer
💻 Technical Questions▼
QuestionKey Points in Your Answer
What's the difference between RANK and DENSE_RANK?RANK skips next number after ties; DENSE_RANK doesn't skip. Show example: 1,2,2,4 vs 1,2,2,3
How do you handle missing data?Identify extent → understand why → choose method (drop/fill mean/median/mode/forward fill) → document decision
What is a p-value?Probability of observing results at least as extreme as yours, assuming null hypothesis is true. <0.05 → reject null
INNER JOIN vs LEFT JOIN?INNER: only matching rows from both. LEFT: all from left table + matching from right (NULL where no match)
How do you detect outliers?IQR method (Q1-1.5×IQR, Q3+1.5×IQR), Z-score (>3 std devs), visualization (box plot, scatter)
What is A/B testing?Controlled experiment with two variants. Define hypothesis → split users → run test → check statistical significance → decide
